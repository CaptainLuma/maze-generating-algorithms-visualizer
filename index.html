Recursive Backtracking / Depth First Search

  1. Set any cell to the active cell.
  2. Repeat the following:
    1. Branch from the active cell to a random neighboring cell that has not yet been visited. This becomes the new active cell.
    2. If all neighboring cells have been visited, travel to the previously visited cell.
  3. The maze is complete once the active cell returns to the first activated cell.

Origin Shift / Reverse Aldous Broder

  1. Start with any perfect maze. The maze must be a root directed tree, meaning each node must point to another node, with a single node pointing nowhere.
  2. Repeat for as many times as you'd like:
    1. have the origin (root) node point to a random neighboring node.
    2. Remove the neighboring nodes pointer and call it the new root node.

Prim's Algorithm

  1. Mark any cell as visited.
  2. Repeat until all cells are visited:
    1. Pick a random visited cell that has at least one unvisited neighbor.
    2. From this cell, branch to a random unvisited neighbor. Add this neighbor to the list of visited cells.

Hunt And Kill Algorithm

  1. Set any cell to the active cell.
  2. Repeat the following:
    1. If not in hunt mode: Branch from the active cell to a random neighboring cell that has not yet been visited. This becomes the new active cell. If all neighboring cells have been visited, enter "hunt" mode.
    2. If in hunt mode: Scan through each cell in the grid until a visited cell with at least one unvisited neighbor is found. Set this to the new active cell, and exit hunt mode. If a valid cell is not found, the maze is complete.

Kruskal's Algorithm

  1. Initialize each node to be a part of their own set.
  2. Repeat the following for every connection in random order:
    1. If the two nodes for this connection are part of different sets, connect the nodes and join their sets.
    2. Do nothing if the nodes are already part of the same set.

Aldous Broder

  1. Set any cell to the active cell
  2. Repeat the following:
    1. Move the active cell to a random neighboring cell
    2. If the new cell hasn't been visited before, carve a path to it from the previous active cell.
  3. Conclude the algorithm once every cell has been visited by the active cell.
